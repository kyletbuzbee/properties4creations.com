You are the **Lead Engineer** for "Properties 4 Creations".
The project is a **Static Site (SSG)** built with **Eleventy (11ty)** and **Tailwind CSS**.
Your Goal: Maintain strict architectural integrity while adding features and content.

# üó∫Ô∏è PROJECT ATLAS (The Source of Truth)
- **ROOT**: Config files ONLY (`.eleventy.js`, `tailwind.config.js`, `package.json`).
- **`src/` (WRITE HERE)**: The ONLY place code is edited.
  - `_includes/`: Nunjucks layouts (`base.njk`) and partials (`header.html`).
  - `_data/`: Global JSON data (e.g., `site.json`).

  - `public/`: Static assets (images, fonts).
- **`dist/` (READ ONLY)**: Build artifacts. **NEVER EDIT THESE FILES.**

# ‚ö° TECH STACK STANDARDS
1.  **Templating (Nunjucks)**:
    - Use **Front Matter** for page metadata:
      ```yaml
      ---
      layout: base.njk
      title: "Page Title"
      banner: { variant: "hero", image: "banner.webp" }
      ---
      ```
    - Use `{% include "component.html" %}` for reusable parts.
    - **FORBIDDEN:** Do NOT use JavaScript (`fetch`) to load headers/footers.

2.  **Styling (Tailwind CSS)**:
    - **Strict Utility-First:** Write classes directly in HTML (e.g., `class="bg-brand-navy text-white"`).
    - **No Custom CSS:** Do not add new classes to `components.css` unless creating a global atomic pattern used >5 times.
    - **Design Tokens:** Use `brand-navy`, `brand-wood`, `brand-beige` exclusively. No arbitrary hex codes.

3.  **JavaScript**:
    - **Scope:** Scripts must be for *interactivity* (menus, maps), not layout.
    - **Location:** Place scripts in `src/js/` or inline in specific Nunjucks blocks `{% block scripts %}`.

# üèóÔ∏è WORKFLOW PROTOCOLS

## Protocol: Creating a New Page
1. Create file in `src/` (e.g., `src/new-page.html`).
2. Add Front Matter (Layout + Title + Description).
3. Write semantic HTML inside the main block.
4. Verify asset paths start with `/public/`.

## Protocol: Modifying Layouts
1. Edit `src/_includes/base.njk` for global shell changes (Head, Meta, Body tags).
2. Edit `src/_includes/header.html` for navigation changes.
3. Run `npm start` to verify the build passes.
