Image Optimization Examples for P4C

This page shows best practices for implementing optimized images with WebP support.

Example 1: Hero Image (With Responsive Sizes)

Location: Top of home page

Current: Single large JPEG (2-3 MB)

Optimized: Multiple WebP sizes (200-400 KB total)

HTML Code:

<picture> <source srcset="public/images/hero-320w.webp 320w, public/images/hero-640w.webp 640w, public/images/hero-960w.webp 960w, public/images/hero-1280w.webp 1280w, public/images/hero-1920w.webp 1920w" type="image/webp"> <source srcset="public/images/hero-320w.jpg 320w, public/images/hero-640w.jpg 640w, public/images/hero-960w.jpg 960w, public/images/hero-1280w.jpg 1280w, public/images/hero-1920w.jpg 1920w" type="image/jpeg"> <img src="public/images/hero-1280w.jpg" alt="Beautiful modern home exterior with landscaping" loading="lazy" decoding="async" style="width: 100%; height: auto; display: block;"> </picture>

What This Does:

  • Mobile (320px): Loads smallest image (hero-320w.webp)
  • Tablet (640px): Loads medium image (hero-640w.webp)
  • Desktop (1280px): Loads large image (hero-1280w.webp)
  • 4K (1920px): Loads largest image (hero-1920w.webp)
  • Old browsers: Falls back to JPEG versions
  • Very old browsers: Falls back to single JPG
  • Lazy loading: Loads only when scrolled into view
  • Async decoding: Browser decodes image without blocking

Example 2: Project/Property Image (Single Size)

Location: Projects page, property details

Current: Single JPEG per project

Optimized: Single WebP with JPEG fallback

HTML Code:

<picture> <source srcset="public/images/projects_finished_wood_deck.webp" type="image/webp"> <source srcset="public/images/projects_finished_wood_deck.jpg" type="image/jpeg"> <img src="public/images/projects_finished_wood_deck.jpg" alt="Finished custom wood deck with seating area" loading="lazy" decoding="async" style="width: 100%; height: auto;"> </picture>

Simpler Version (If You Prefer):

<img src="public/images/projects_finished_wood_deck.webp" alt="Finished custom wood deck with seating area" loading="lazy" decoding="async" style="width: 100%; height: auto;">

Note: The simpler version works if most users have modern browsers. Use the picture version for maximum compatibility.

Example 3: Thumbnail/Small Image

Location: Project cards, property listings

Current: JPEG thumbnail (50-150 KB)

Optimized: WebP thumbnail (10-30 KB)

HTML Code:

<img src="public/images/projects_wood_deck.webp" alt="Wood deck project thumbnail" width="300" height="200" loading="lazy" decoding="async">

Why simpler: Thumbnails are already small, don't need responsive sizes. WebP alone provides 60-70% size reduction.

Example 4: Image Gallery (Multiple Images)

Location: Before/After gallery, property details

Pattern: Repeat picture element for each image

HTML Code:

<div class="gallery"> <picture> <source srcset="public/images/before-after/deck-before.webp" type="image/webp"> <source srcset="public/images/before-after/deck-before.jpg" type="image/jpeg"> <img src="public/images/before-after/deck-before.jpg" alt="Wood deck before project" loading="lazy" decoding="async"> </picture> <picture> <source srcset="public/images/before-after/deck-after.webp" type="image/webp"> <source srcset="public/images/before-after/deck-after.jpg" type="image/jpeg"> <img src="public/images/before-after/deck-after.jpg" alt="Wood deck after project completion" loading="lazy" decoding="async"> </picture> </div>

Example 5: Icon or SVG Image

Location: Icons, logos, decorative elements

Note: SVGs are already optimized, no need to convert to WebP

HTML Code:

<!-- SVG (No conversion needed) --> <img src="public/images/logo/p4c-logo.svg" alt="P4C Logo" style="width: 100px; height: auto;"> <!-- Raster Icon (Convert to WebP) --> <img src="public/images/icons/hammer.webp" alt="Hammer icon" style="width: 32px; height: 32px;">

Best Practices Checklist

  • Always include alt text (describes image for screen readers)
  • Use loading="lazy" for below-fold images
  • Use decoding="async" for non-critical images
  • For mobile-first: start with smallest image
  • Test in browser DevTools (Network tab shows which image loaded)
  • Use WebP with JPEG fallback for maximum compatibility
  • Don't convert SVGs to WebP (they're already optimized)
  • Test on actual phone (Chrome DevTools is limited)
  • Check file sizes reduced 60-75% from originals
  • Run Lighthouse audit after updates

Testing Your Images

Step 1: Check DevTools (Chrome)

1. Open page in Chrome 2. Press F12 (Open DevTools) 3. Go to Network tab 4. Reload page 5. Filter by "img" to see images 6. Check if WebP files are loaded (not JPEG) 7. Check file sizes (should be 60-70% smaller) 8. Scroll to bottom - images should load on scroll

Step 2: Check Lazy Loading

1. Open DevTools Network tab 2. Reload page 3. WITHOUT scrolling, notice which images loaded 4. Scroll down slowly 5. Watch for more images loading as you scroll 6. If images load before scroll = not lazy loading

Step 3: Run Lighthouse

1. Open page in Chrome 2. Press F12 3. Go to Lighthouse tab 4. Click "Analyze page load" 5. Score should be 90+ (or improved from before) 6. Look for "Offscreen images" - if none, lazy loading works
